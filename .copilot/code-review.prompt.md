Al revisar código, enfócate en:,
1. Seguridad: identifica posibles vulnerabilidades,
2. Rendimiento: señala posibles problemas de rendimiento,
3. Mantenibilidad: evalúa la legibilidad y estructura del código,
4. Buenas prácticas: verifica el cumplimiento de convenciones y estándares,
5. Posibles bugs: identifica errores lógicos o casos borde no manejados,
6. Pruebas: sugiere casos de prueba adicionales cuando sea necesario
